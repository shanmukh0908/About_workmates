Security Architecture

JWT Authentication with Access and Refresh Tokens

Implemented authentication using short-lived access tokens and HTTP-only refresh tokens to keep users loggedin , maintains security.

more... ✖
Implementation Approach:
When an access token expires, the server checks the refresh token stored in a secure cookie. If valid, a new access token is generated and sent back to the client automatically.

Password updates invalidate previous tokens using a passwordChangedAt timestamp comparison.

Result:
• Secure session handling using HTTP-only cookies
• Users remain logged in without repeated login prompts
• Token expiration handled automatically
JWT • HTTP-only Cookies • Refresh Tokens • Express Middleware
Performance Optimization

Direct Image Upload to Cloudinary storage from frontend

Improved the image upload process by allowing the frontend to upload images directly to Cloudinary using signed response with trasnformation rule and other rules.

more... ✖
Implementation Approach:
Instead of sending images through the backend, the server generates a signed upload token and the React frontend uploads files directly to Cloudinary.

Multiple images are uploaded concurrently using Promise.all(), while Cloudinary handles automatic resizing and compression.

Result:
• Reduced backend load and memory usage
• Faster uploads for users
• Simpler backend image handling
Cloudinary API • Node.js • Promise.all • FormData
Geo-Spatial System

Location Based Search with 2dsphere Index

Built a location-aware task search system using MongoDB 2dsphere indexes to find tasks and workmates near the user.

more... ✖
Implementation Approach:
Applied 2dsphere indexes on user and task locations to enable efficient radius-based queries using $near.

Distance from the user is calculated using geolib to display clear distance information in the UI.

Result:
• Faster location searches using indexing
• Ability to filter nearby tasks and providers
• Improved location-based user experience
MongoDB • 2dsphere Index • $near • Geolib

Data Integrity

MongoDB Session-Based Transactions

Used MongoDB transactions to ensure multiple related database operations either complete together or roll back on failure.

more... ✖
Implementation Approach:
Wrapped related database updates inside a MongoDB session using startTransaction → commitTransaction → abortTransaction.

If any step fails, the transaction is aborted to avoid partial updates across collections.

Result:
• Consistent database state
• No partial writes
• Safer multi-step operations
MongoDB Transactions • Mongoose Sessions • Atomic Writes

Automatic Rating Calculation

more... ✖
